# Hormon Care - Patient App (React Native CLI)

A native Android/iOS mobile app for **Hormon Care - Gynecology Patient Care**.
The app connects directly to the live production backend at
`https://hormoncare.mediiqr.com` (the same database that powers the web
patient portal). All patient data, plans, weekly followups and lifestyle
assessments are synced in real time with the existing web system.

## Features

- 🔐 **Patient Login** – Patient ID + password (same credentials as web)
- 🏠 **Dashboard** – patient name, assigned plan card, weekly progress
- 📅 **Plan Weeks** – view every week with YouTube videos, text notes, images
- 📋 **Lifestyle Assessment** – full 9-section form (Physical Activity, BMI,
  Sleep, Stress Screening, Diet, Habits, Medical History, Family History,
  Partner Lifestyle) with automatic BMI calculation
- 📝 **Weekly Followup** – submit weekly check-in (weight, exercise days,
  water, sleep, supplements, meal deviations, plan feedback)
- 🚪 Session persistence via `AsyncStorage`, pull-to-refresh, error banners

## Backend / API

The app is a **thin client** – it talks straight to the existing Next.js
backend that already powers the web app. Endpoints used:

| Method | Endpoint | Purpose |
|--------|-----------|---------|
| POST | `/api/auth/patient/login` | Login → sets `session` cookie |
| POST | `/api/auth/logout` | Logout |
| GET  | `/api/patient/dashboard` | Profile + plan + weeks + contents |
| GET  | `/api/patient/gate-status` | Blocking prompts (lifestyle / followup) |
| GET  | `/api/patient/lifestyle-assessment` | Assessment status |
| POST | `/api/patient/lifestyle-assessment` | Submit assessment |
| GET  | `/api/patient/followup?week=N` | Load followup for week |
| GET  | `/api/patient/followup/status` | Pending / due weeks |
| POST | `/api/patient/followup` | Submit weekly followup |

The Next.js server issues an **HttpOnly** `session` JWT cookie. React
Native's fetch does not persist cookies automatically, so the client
extracts the raw `Set-Cookie` header at login and re-sends it as `Cookie`
on every subsequent request. The cookie is stored in `AsyncStorage`, so
users stay logged in across app restarts.

## Prerequisites (local machine)

Because this is **React Native CLI** (not Expo), you need a real dev
environment to build/run:

1. **Node.js 22.11+** and **Yarn**
2. **JDK 17** (Android)
3. **Android Studio** with an SDK + emulator (or a real device with USB
   debugging)
4. **Xcode** (only if you want to build for iOS on macOS)

Follow the official React Native "Environment Setup" guide:
<https://reactnative.dev/docs/set-up-your-environment>

## Run the App

```bash
# 1. Install JS dependencies
cd HormonCarePatient
yarn install

# 2. iOS pods (macOS only)
cd ios && pod install && cd ..

# 3. Start Metro bundler
yarn start

# 4. In another terminal, run on Android
yarn android

# ...or on iOS (macOS only)
yarn ios
```

## Build a Release APK (Android)

```bash
cd android
./gradlew assembleRelease
# APK will be at android/app/build/outputs/apk/release/app-release.apk
```

For a signed release you'll need to configure a keystore in
`android/app/build.gradle` and `android/gradle.properties`.

## Notes

- The app reads/writes to the **live production database** through the
  existing backend — there is no separate mobile-only backend to maintain.
- All Gujarati/English copy is inlined; the app defaults to Gujarati to
  match the web experience.
- YouTube plan videos are rendered in an embedded WebView. Full-screen
  playback is supported.
- No local database — dashboard is reloaded on every launch (pull-to-refresh
  to force reload).
